# NeuroX v7.4: Platt calibration, micro-pullback entry, Sharpe-based model weights

NeuroX v4 is a full fork of the existing neurox/ codebase into a new `neurox_v4/` directory, adding three features to the signal pipeline: Platt scaling confidence calibration that adjusts raw model confidence using a rolling sigmoid fit, a micro-pullback entry timer that delays trade execution by up to 10 seconds to catch a better price, and Sharpe-ratio-based model weighting that replaces the win-rate-only dynamic weights with risk-adjusted returns. The fork-as-version approach keeps v4 entirely independent of the original codebase. The MQ5 EA and batch launcher are also versioned.

**Watch for:** Entry timing has a signal-loss path where a pending pullback triggers on a HOLD cycle but never re-emits the signal to the bridge (confirmed). Sharpe weighting attributes PnL equally to all 17 models regardless of which one drove the prediction, diluting the signal (confirmed). No dedicated unit tests exist for any of the three new features (confirmed).

## High-level view

Platt scaling fits a sigmoid (A, B) on a rolling window of 200 (logit, win/loss) pairs using Nelder-Mead optimization, then remaps raw confidence before it reaches the Brain and bridge. The calibrator persists state to a JSON file across restarts, so the warm-up penalty (50-sample minimum) is paid only once. The state file path is relative to CWD, which can break if the process launches from a different directory.

Micro-pullback entry registers a pending signal when a non-HOLD signal fires, then on each subsequent cycle checks whether price has pulled back by $0.30 (gold), broken out by $1.00 in the adverse direction, or timed out at 10 seconds. The HOLD-cycle re-trigger path logs the trigger without actually writing to the bridge, meaning a well-timed pullback during a HOLD cycle is silently dropped.

Sharpe weighting maintains a per-model rolling deque of PnL values and recomputes weights every 20 trades once at least 3 models have 20+ samples. The floor mechanism (2% minimum per model) prevents zero allocation. However, PnL is attributed to every model equally on each trade close, not to the model(s) whose prediction drove the actual trade. This makes all 17 Sharpe curves converge to the portfolio-level Sharpe, defeating the purpose of per-model differentiation.

## Entry timing signal loss on HOLD-cycle re-trigger

The HOLD-cycle branch (line ~735 of `main.py`) detects that a pending signal has timed out or pulled back enough, logs "Pending signal triggered," but the `signal` variable remains HOLD. Since the bridge write is gated on `signal.action != "HOLD"`, nothing is sent to MT5:

```python
elif self.entry_timing and signal.action == "HOLD":
    if self.entry_timing.has_pending_signal:
        status = self.entry_timing.get_status()
        entry_result = self.entry_timing.evaluate_entry(
            action=status['action'],
            current_price=current_price,
        )
        if entry_result['should_enter']:
            # Logs but never mutates signal or writes to bridge
            self.logger.info(
                f"[EntryTiming] Pending signal triggered: {entry_result['reason']}"
            )
```

The comment says "the bridge write happens next cycle," but by then the pending state has been cleared (the `evaluate_entry` call internally runs `_clear_pending`) and the next cycle's signal generator may not regenerate the same signal. This is a confirmed missed-trade path.

## Sharpe weighting: undifferentiated PnL attribution

The trade-close handler in the confirmation poller:

```python
for mn in ensemble._model_names:
    ensemble.update_pnl_attribution(mn, pnl)
ensemble.sharpe_reweight()
```

Every model receives the same PnL for every trade. After 100+ trades, all 17 models will have nearly identical mean and std, producing identical Sharpe ratios. The `compute_sharpe_weights` method works correctly in isolation — the issue is entirely in how it's fed data. The ensemble's `predict` method already returns `individual_preds` per model, which could be used to identify which models voted for the winning direction and attribute PnL proportionally.

## Platt calibration: permissive convergence check

The success check (`result.success or result.fun < 10.0`) accepts a NLL of 10.0 on binary classification — effectively random noise territory for a sigmoid fit. A tighter bound (e.g., < 0.7, the entropy of a fair coin) would prevent the calibrator from applying a meaningless transform. Additionally, the state save (`save_state()`) runs synchronously on the confirmation poller thread; if disk I/O stalls during heavy confirmation traffic, it delays position sync processing.

## Batch script downloads from hardcoded branch

`neurox_run_v4.bat` downloads source files from `feature/5-model-ensemble-tcn-lgbm` branch on GitHub. There's no checksum or HTTP status validation on the `urllib.request.urlretrieve` calls. If the branch is renamed or merged, the script silently writes 404 HTML pages as `.py` files, which will produce cryptic import errors on startup. Since this is meant as a stable v4 snapshot, the URLs should reference a tag or commit SHA.
