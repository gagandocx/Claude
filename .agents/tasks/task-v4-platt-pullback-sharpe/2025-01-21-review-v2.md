# NeuroX v7.4: Platt calibration, micro-pullback entry, Sharpe-based model weights (v2)

The three v1 blocking issues — HOLD-cycle signal loss, undifferentiated Sharpe attribution, and the relative Platt state path — have been fixed. Entry timing now stores `signal_details` and reconstructs a full `TradeSignal` for the bridge write. Sharpe attribution now distributes PnL proportionally among models whose directional probability exceeded 1/3. The Platt state file uses an absolute path anchored to `_NEUROX_V4_ROOT`.

**Watch for:** Entry timing's HOLD-cycle path clears `_active_position` before the signal triggers, so trades that enter via pullback have no position tracking — PnL attribution, Platt recording, and performance metrics are all skipped for those trades (confirmed). Platt calibration's NLL threshold of 10.0 is unreachable for mean binary cross-entropy, making the convergence check a no-op (confirmed). No unit tests for the three new features (unaddressed from v1). Batch script still downloads from an ephemeral branch (unaddressed from v1).

**Verdict**: NEEDS_CHANGES

## High-level view

The HOLD-cycle entry timing fix correctly reconstructs the signal and writes it to the bridge, but it does not re-establish `signal_generator._active_position`. Since the confirmation poller checks that field to find the trade's direction, entry price, and individual model predictions, every trade that enters via the delayed pullback path becomes invisible to the Sharpe tracker, the Platt calibrator, and the performance tracker. For a 10-second timeout with a 2-second loop interval, this could be the common path — any signal that doesn't get its $0.30 pullback within a single iteration will trigger on a subsequent HOLD cycle.

Additionally, `signal_details` (stored in entry_timing for reconstruction) does not include `individual_preds`, so even if `_active_position` were re-created, the per-model prediction data needed for Sharpe attribution would be missing.

The Platt calibrator's convergence check (`result.success or result.fun < 10.0`) is effectively unconditional. Mean binary cross-entropy has a theoretical maximum of ~0.693 per sample, so the 10.0 threshold is unreachable. The check reads as a safety gate but provides no protection against pathological fits on degenerate data.

## HOLD-cycle entries bypass position tracking

When entry timing delays a signal, line 760 clears the active position to unblock single-position enforcement:

```python
signal = type(signal)(
    timestamp=signal.timestamp, symbol=signal.symbol, action="HOLD",
    ...
)
self.signal_generator.clear_active_position()
```

When the HOLD-cycle path later fires and reconstructs the signal, it writes to the bridge but never sets `_active_position`. The confirmation poller's CLOSED handler falls into the "no active tracked" branch:

```python
else:
    self.logger.info(
        f"[INSTANT SYNC] EA closed position #{ticket} "
        f"P&L=${...} (no active tracked)")
```

Everything downstream — Platt recording, Sharpe attribution, performance tracking, brain learning — is gated on the `if active:` condition that just failed.

A second issue compounds this: `signal_details` passed to entry_timing captures confidence, sl_pips, tp_pips, lot_size, model_name, and regime, but does NOT capture `individual_preds`. Even if the fix re-creates `_active_position` in the HOLD-cycle path, it won't have per-model prediction data for Sharpe attribution. The fix at line 733 should include `individual_preds` from the just-cleared `_active_position` before it's wiped:

```python
self.entry_timing.set_pending_signal(
    action=signal.action,
    price=current_price,
    signal_details={
        "confidence": signal.confidence,
        ...
        "individual_preds": self.signal_generator._active_position.get("individual_preds", {}),
    },
)
```

## Platt calibration: convergence threshold semantics

Binary cross-entropy for N samples computes the *mean* NLL, so even worst-case data produces values well below 1.0. A threshold of 10.0 is unreachable under normal conditions. The `if result.success or result.fun < 10.0` branch always executes, which is equivalent to unconditionally accepting the Nelder-Mead result. If the optimizer wanders to a pathological (A, B) pair on degenerate data (e.g., all outcomes = 1), those parameters will be stored and applied. The practical risk is low because 50+ diverse trading outcomes produce a smooth surface, but the code communicates a safety contract it doesn't deliver.
